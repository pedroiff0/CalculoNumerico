# Contribuindo (PT-BR)

Obrigado por querer contribuir com este projeto! Algumas diretrizes rápidas:

- Abra uma *issue* antes de iniciar grandes mudanças para discutir o design.
- Crie uma branch baseada em `main` nomeada com o prefixo `feature/` ou `fix/`.
- Escreva código legível com testes e documentação.
- Docstrings devem seguir o estilo **NumPy** (ex.: parâmetros, retornos, exemplos).
- Rode os testes localmente: `pytest -q`.
- Atualize a documentação (Sphinx) em `docs/` quando adicionar/alterar APIs.

Ao abrir o PR, descreva as mudanças e adicione screenshots ou saída de teste quando relevante.
