Equações Diferenciais Ordinárias (EDOs)

Documentação das funções do módulo codigos.edos relacionadas à resolução de equações diferenciais ordinárias usando métodos numéricos como Runge-Kutta e Euler.

Documentação

As funções listadas abaixo implementam métodos numéricos para resolver EDOs de primeira e segunda ordem, sistemas de EDOs, com opções para diferentes ordens de Runge-Kutta e visualização de resultados.

codigos.edos.runge_kutta(func_input, x0, y0, h, xn, ordem)[código-fonte]

Resolve EDOs escalares usando métodos de Runge-Kutta de ordem 1 a 4.

Parâmetros:

  • func_input (str) – Expressão para f(x, y) como string (ex.: 'y' para dy/dx = y).

  • x0 (float) – Ponto inicial.

  • y0 (float) – Valor inicial y(x0).

  • h (float) – Tamanho do passo (deve dividir o intervalo (xn-x0)).

  • xn (float) – Ponto final onde se integra até (xn > x0 esperado).

  • ordem (int) – Ordem do método (1, 2, 3 ou 4).

Retorna:

Tupla (x_vals, y_vals) com os pontos de amostragem e as aproximações y.

Tipo de retorno:

(list, list)

codigos.edos.runge_kutta_sistema(funcs_input, u0, t0, tf, h, ordem)[código-fonte]

Resolve sistemas de EDOs usando Runge-Kutta (vetorial).

Parâmetros:

  • funcs_input (list of str or callables) – Lista de expressões/funcões que descrevem cada componente do sistema. Use a notação y[1], y[2] etc. quando passar strings (1-based index).

  • u0 (array_like) – Vetor de condições iniciais (tamanho n, onde n é número de equações).

  • t0 (float) – Tempo inicial.

  • tf (float) – Tempo final para integração.

  • h (float) – Passo de tempo.

  • ordem (int) – Ordem do método de Runge-Kutta (1-4).

Retorna:

Tupla (t_vals, u_vals) onde t_vals é lista de tempos e u_vals é array (m x n) com as soluções em cada tempo.

Tipo de retorno:

(list, ndarray)

codigos.edos.executar_sistema_edos()[código-fonte]

Interface para resolução de sistemas de EDOs usando Runge-Kutta.

codigos.edos.executar_edo_2ordem()[código-fonte]

Interface para resolução de EDOs de 2ª ordem convertidas em sistemas.

Exemplo de Uso

 1"""Exemplo: resolver EDOs com Runge-Kutta (scalar e sistemas).
 2
 3Executar: python tests/exemplos/exemplo_edos.py
 4"""
 5from codigos import EDOs
 6import numpy as np
 7
 8if __name__ == '__main__':
 9    # Exemplo escalar: dy/dx = y, y(0)=1
10    x_vals, y_vals = EDOs.runge_kutta('y', 0.0, 1.0, 0.1, 1.0, 4)
11    print('RK4 escalar: y(1) ≈', y_vals[-1], 'exato e ≈', np.exp(1.0))
12
13    # Exemplo de sistema
14    funcs = ['y[1]', '2*y[2]']
15    t_vals, u_vals = EDOs.runge_kutta_sistema(funcs, np.array([1.0, 1.0]), 0.0, 1.0, 0.1, 4)
16    print('RK4 sistema: u(1) ≈', u_vals[-1], 'exato [e, e^2] ≈', np.exp(1.0), np.exp(2.0))

Testes Unitários (visíveis)

Abaixo está o arquivo de testes unitários para a funcionalidade de EDOs. Ele é incluído para fácil visualização e referência.

 1import numpy as np
 2import pytest
 3import math
 4from codigos import edos
 5
 6def test_runge_kutta_scalar_exp():
 7    # dy/dx = y, y(0)=1 -> y(1)=e
 8    x_vals, y_vals = edos.runge_kutta('y', 0.0, 1.0, 0.01, 1.0, 4)
 9    approx = y_vals[-1]
10    assert abs(approx - np.exp(1.0)) < 5e-4
11
12
13def test_runge_kutta_system_exp():
14    # system: u0' = u0, u1' = 2*u1 with u0(0)=u1(0)=1
15    funcs = ['y[1]', '2*y[2]']
16    # usar passo menor para alcançar precisão suficiente em testes automatizados
17    t_vals, u_vals = edos.runge_kutta_sistema(funcs, np.array([1.0, 1.0]), 0.0, 1.0, 0.001, 4)
18    u_end = u_vals[-1]
19    assert abs(u_end[0] - np.exp(1.0)) < 1e-4
20    assert abs(u_end[1] - np.exp(2.0)) < 1e-3

23def test_passos_edo_with_m():

29def test_runge_kutta_orders_accuracy():
30    # dy/dx = y, y(0)=1 -> y(1) = e
31    exact = math.e
32    h = 0.001
33    tol_order = {1:5e-3, 2:1e-4, 3:5e-5, 4:1e-6}
34    for order, tol in tol_order.items():
35        x_vals, y_vals = edos.runge_kutta('y', 0.0, 1.0, h, 1.0, order)
36        approx = y_vals[-1]
37        assert abs(approx - exact) < tol

45def test_runge_kutta_sistema_last_step():
46    # system: u0' = u0, u1' = u1, with initial [1,1], exact at t=1 => [e, e]
47    funcs = ['y[1]', 'y[2]']
48    t_vals, u_vals = edos.runge_kutta_sistema(funcs, np.array([1.0,1.0]), 0.0, 1.0, 0.3, 4)
49    # ensure final time equals tf exactly (last step smaller than h)
50    assert pytest.approx(t_vals[-1], rel=1e-12) == 1.0

53def test_runge_kutta_sistema_predator_prey():
54    # Lotka-Volterra: u' = u - u*v, v' = -v + u*v with u(0)=2, v(0)=1
55    funcs = ['y[1] - y[1]*y[2]', '-y[2] + y[1]*y[2]']
56    t_vals, u_vals = edos.runge_kutta_sistema(funcs, np.array([2.0, 1.0]), 0.0, 0.5, 0.01, 4)
57    # Check that values are positive and reasonable
58    assert all(u > 0 for u in u_vals.flatten())
59    assert u_vals[0][0] == 2.0  # initial
60    assert u_vals[0][1] == 1.0
